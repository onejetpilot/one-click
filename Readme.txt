Проект: Обучение с учителем: качество модели

Описание проекта

Интернет-магазин «В один клик» продаёт разные товары: для детей, для дома, мелкую бытовую технику, косметику и даже продукты. Отчёт магазина за прошлый период показал, что активность покупателей начала снижаться. Привлекать новых клиентов уже не так эффективно: о магазине и так знает большая часть целевой аудитории. Возможный выход — удерживать активность постоянных клиентов. Сделать это можно с помощью персонализированных предложений.

«В один клик» — современная компания, поэтому её руководство не хочет принимать решения просто так — только на основе анализа данных и бизнес-моделирования. У компании есть небольшой отдел цифровых технологий, и вам предстоит побыть в роли стажёра в этом отделе. 

Задача проекта

Разработать решение, которое позволит персонализировать предложения постоянным клиентам, чтобы увеличить их покупательскую активность.

При выполнении проекта провели следующую работу:
    
- Проверили типы данных, дубликаты и пропущенные значения в датасетах
- Провели исследовательский анализ данных
- Объединили датафремы m_file m_money m_time 
- Построили матрицу корреляции
- Сделали пайплайн обработки данных и обучения модели
- Построили график значимости признаков c помощью метода SHAP
- Описали признаки
- Выделели и проанализировали сегмент пользователей

Построение пайплайна и выбор модели:

- Для предобработки числовый признаков модели лучше всего показал себя MinMaxScaler().
- Для предобработки категориальных признаков использовался OrdinalEncoder и OheHotEncoder.
- Для оценки качества модели была выбрана метрика recall. 
- Лучше всего показала себя модель LogisticRegression с гиперпараметрами (random_state=42, solver='saga')
- Была получена метрика ROC-AUC на тестовой выборке для 1 класса - 0.92.
